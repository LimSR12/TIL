# 🐳 Docker란 무엇인가

## 1. Docker의 정의

**Docker(도커)** 는 애플리케이션을 컨테이너라는 단위로 패키징하여 어디서나 동일한 환경에서 실행할 수 있도록 도와주는 컨테이너 기반 가상화 플랫폼이다.

> 환경 차이로 인한 문제를 없애고, 애플리케이션 실행 환경을 표준화해주는 도구

## 2. Docker가 등장한 배경

개발 환경과 실행 환경이 달라 발생하는 대표적인 문제들:

- 로컬에서는 잘 되는데 서버에서는 안 됨
- 팀원마다 OS, 라이브러리, 버전이 다름
- 서버에 직접 설치하다가 환경이 망가짐
- 배포할 때마다 설정을 다시 해야 함

Docker는 이런 문제를 다음 방식으로 해결한다.

- 애플리케이션 + 실행에 필요한 모든 의존성을 하나의 이미지(Image) 로 묶음
- 어디서 실행하든 동일한 결과 보장

## 3. Docker의 핵심 개념

### 3.1 컨테이너

- 애플리케이션과 그 실행 환경을 함께 담은 실행 단위
- OS 전체를 가상화하지 않고, 프로세스 단위로 격리
- 가볍고 실행 속도가 빠름

#### 📌 특징

- 독립적인 파일 시스템
- 독립적인 네트워크
- 호스트 OS 커널을 공유

### 3.2 이미지(Image)

- 컨테이너를 만들기 위한 설계도
- 실행 환경 + 애플리케이션이 이미 정의되어 있음
- 불변(Immutable) → 수정 시 새 이미지 생성

예시:

- node:20-alpine
- mysql:8.0
- nginx:latest

### 3.3 Dockerfile

- 이미지를 생성하기 위한 설정 파일
- 어떤 환경에서, 어떤 명령을 실행할지 코드로 정의

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY . .
RUN npm install
CMD ["npm", "start"]
```

#### 📌 의미

→ "이 애플리케이션은 Node 20 환경에서 이렇게 실행된다"

### 3.4 Docker Engine

- Docker의 핵심 실행 엔진
- 이미지 다운로드, 컨테이너 생성/실행/삭제를 담당
- 보통 docker 명령어를 통해 제어

## 4. Docker vs 가상머신(VM)

| 구분        | Docker (Container)   | VM        |
| ----------- | -------------------- | --------- |
| 가상화 대상 | 프로세스             | OS 전체   |
| 속도        | 매우 빠름            | 느림      |
| 용량        | 가벼움               | 무거움    |
| 실행 환경   | 호스트 커널 공유     | 독립 커널 |
| 활용        | 마이크로서비스, 배포 | 서버 격리 |

📌 Docker는 가볍고 빠른 배포에 최적화된 방식

## 5. Docker의 장점

- ✅ 개발/운영 환경 일관성 보장
- ✅ 빠른 실행 및 배포
- ✅ 설정을 코드로 관리 가능 (Infrastructure as Code)
- ✅ 로컬 ↔ 서버 ↔ CI/CD 환경 통일
- ✅ 마이크로서비스 아키텍처에 적합

## 6. Docker의 한계 및 주의점

- OS 커널을 공유하므로 완전한 OS 격리는 아님
- 보안 설정을 제대로 하지 않으면 위험
- 컨테이너 수가 많아지면 관리 복잡도 증가
  → 이 문제를 해결하기 위해 Docker Compose, Kubernetes 등장

## 7. 언제 Docker를 사용하는가

- 개발 환경을 팀원 간 동일하게 맞추고 싶을 때
- 로컬에서 서버 환경을 그대로 재현하고 싶을 때
- CI/CD 파이프라인에서 동일한 실행 환경이 필요할 때
- 마이크로서비스 구조로 서비스를 운영할 때
